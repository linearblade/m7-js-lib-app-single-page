Page 1: Crisp Content Swap

This page is intentionally text-heavy enough that a partial swap is obvious. It is a good route for confirming that the old content is fully replaced rather than merged accidentally.

What to watch

Primary check

Content integrity

Move from this page to Page 2, then hit Back. The wording, panel order, and metrics here should return exactly as they were, which makes it easy to notice stale fragments or a bad restore sequence.

Secondary check

History behavior

  • The URL should update without a full reload.
  • The title should follow the page you moved to.
  • Back should restore this scene without writing a new history entry.
Swap target

Only #main changes, which keeps the ticker and navigation chrome stable across routes.

Handler pairing

The click side writes the route meaning, and the popstate side restores it when the browser drives navigation.

Good canary

If something regresses in content replacement, this page’s longer copy usually makes it obvious right away.
